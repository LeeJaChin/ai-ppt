## 快速开始

### 方式 1：使用启动脚本（推荐）

#### macOS/Linux

```bash
# 在项目根目录执行
./start.sh
```

#### Windows

```bash
# 在项目根目录双击或执行
start.bat
```

### 方式 2：手动启动

#### 1. 启动后端

```bash
cd backend

# 创建虚拟环境（首次运行）
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate  # macOS/Linux
# 或
venv\Scripts\activate.bat  # Windows

# 安装依赖
pip install -r requirements.txt

# 启动服务
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 2. 启动前端

打开新终端：

```bash
cd frontend

# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

## 配置 API Keys

### 1. 获取 API Keys

您至少需要配置一个 AI 模型的 API Key：

#### OpenAI GPT-4o

1. 访问 [OpenAI Platform](https://platform.openai.com/)
2. 登录账号
3. 进入 API Keys 页面
4. 创建新的 API Key
5. 复制密钥（格式：`sk-...`）

#### Anthropic Claude 3.5

1. 访问 [Anthropic Console](https://console.anthropic.com/)
2. 登录账号
3. 进入 API Keys 页面
4. 创建新的 API Key
5. 复制密钥（格式：`sk-ant-...`）

#### DeepSeek

1. 访问 [DeepSeek Platform](https://platform.deepseek.com/)
2. 注册并登录
3. 进入 API Keys 管理
4. 创建新的 API Key
5. 复制密钥

#### Google Gemini

1. 访问 [Google AI Studio](https://makersuite.google.com/app/apikey)
2. 登录 Google 账号
3. 创建 API Key
4. 复制密钥

### 2. 配置密钥

编辑 `backend/.env` 文件：

```env
# 至少配置一个
OPENAI_API_KEY=sk-your-openai-key-here
ANTHROPIC_API_KEY=sk-ant-your-anthropic-key-here
DEEPSEEK_API_KEY=your-deepseek-key-here
GEMINI_API_KEY=your-gemini-key-here
```

**重要**：
- 只配置您拥有的模型密钥即可
- 未配置的模型不会出现在选择列表中
- 密钥不要提交到版本控制

## 使用示例

### 示例 1：生成公司汇报 PPT

**输入需求：**

```
关于人工智能在医疗行业应用的公司汇报

包括以下内容：
- AI 医疗的市场规模和发展趋势
- 核心技术：图像识别、自然语言处理、预测分析
- 典型应用场景：疾病诊断、药物研发、个性化治疗
- 成功案例分析
- 我们的解决方案和产品
- 市场竞争格局
- 未来发展计划
```

**操作步骤：**

1. 将上述文本粘贴到输入框
2. 选择 AI 模型（如 GPT-4o）
3. 选择主题风格（建议选择"商务风格"）
4. 点击"生成大纲"
5. 等待 10-30 秒，AI 生成结构化大纲
6. 在预览区编辑大纲（可选）
7. 点击"确认生成 PPT"
8. 等待生成完成，自动下载

## 常见问题

### Q1: 生成大纲失败，显示 API 错误？

**解决方法：**

1. 检查 `backend/.env` 文件中对应模型的 API Key 是否正确
2. 确认 API Key 有足够的额度
3. 检查网络连接，确保能访问 API 服务
4. 尝试切换其他模型

### Q2: 生成的大纲不符合预期？

**解决方法：**

1. **优化输入**：提供更详细的需求描述
2. **使用结构化输入**：使用列表或大纲格式
3. **指定幻灯片数量**：如"生成 8-10 页 PPT"
4. **尝试不同模型**：不同模型风格不同
5. **编辑大纲**：在预览区手动调整

### Q3: PPT 生成很慢？

**原因：**
- 大纲复杂，幻灯片较多
- 服务器性能限制

**解决方法：**
- 减少幻灯片数量（建议 5-15 页）
- 简化要点内容

### Q4: 下载的 PPT 无法打开？

**检查项：**

1. 文件是否完整下载
2. 使用 Microsoft PowerPoint、WPS 或 LibreOffice 打开
3. 检查后端日志是否有错误

### Q5: 前端连接不到后端？

**解决方法：**

1. 确认后端服务正在运行（访问 http://localhost:8000）
2. 检查防火墙设置
3. 查看浏览器控制台的网络请求
4. 确认 `.env.local` 中的 API_URL 配置正确

## 最佳实践

### 1. 输入需求的技巧

✅ **好的输入：**

```
为初创公司投资人路演准备 PPT

公司：AI 智能客服平台
融资轮次：Pre-A 轮，寻求 500 万美元

需要包含：
- 问题与机会（市场痛点）
- 解决方案（产品介绍）
- 商业模式
- 市场规模和竞争分析
- 核心团队
- 发展历程和里程碑
- 财务预测
- 融资用途
- 联系方式
```

❌ **不好的输入：**

```
做个 PPT
```

### 2. 选择合适的模型

| 场景 | 推荐模型 | 理由 |
|------|---------|------|
| 商业汇报 | GPT-4o | 结构清晰，内容专业 |
| 技术文档 | Claude 3.5 | 技术细节丰富，逻辑严谨 |
| 创意提案 | GPT-4o / Claude 3.5 | 创意性强 |
| 教学课件 | DeepSeek | 内容详细，价格实惠 |

### 3. 选择合适的主题

- **商务风格**：正式汇报、投资路演、年度总结
- **科技风格**：技术分享、产品发布、研发进展
- **创意风格**：营销方案、创意提案、活动策划

### 4. 优化生成效果

1. **明确目标受众**：在需求中说明"面向投资人"、"面向技术团队"等
2. **指定关键信息**：重要数据、案例、时间点等
3. **控制长度**：明确页数要求（如"8-10 页"）
4. **多次尝试**：不满意可重新生成，AI 每次结果略有不同

### 5. 后期处理

生成的 PPT 是可编辑的 .pptx 文件，建议：

1. 使用 PowerPoint 打开并微调
2. 添加公司 Logo 和品牌元素
3. 插入图片、图表等可视化内容
4. 调整字体大小和行距
5. 检查错别字和语法

## 技巧与窍门

### 快速生成系列 PPT

如果需要多个相关 PPT：

1. 准备好多个需求描述
2. 依次生成，保持一致的主题风格
3. 使用相同的模型，保持风格统一

### 创建模板库

1. 生成一个满意的 PPT
2. 在 PowerPoint 中另存为模板 (.potx)
3. 下次基于模板修改，更高效

### 批量处理

对于大量 PPT 需求：

1. 可以通过 API 直接调用（查看 API 文档）
2. 编写脚本自动化处理
